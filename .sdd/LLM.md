# LLM Instructions

You are working in a Spec-Driven Development repository.

## Rules

1. Always read `.sdd/WORKFLOW.md` before implementation.
2. Always read the relevant `SPEC.md` before implementation.
3. Never implement a task if its parent spec is not `approved` or `in-progress`.
4. Follow `PLAN.md`, `TASKS.md`, `DESIGN.md`, and `ARCHITECTURE.md`.
5. Work only on the selected task.
6. Do not modify unrelated files.
7. Report changed files.
8. Report test instructions.
9. If requirements are unclear, update questions in the spec instead of guessing.

## Output format after implementation

- Summary
- Changed files
- How to test
- Risks / assumptions
- Next suggested task
